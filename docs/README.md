## 📊 Project Overview

**ClarityAge - 40** is an interactive web application that presents a philosophical questionnaire to help users reflect on their life perspectives, values, and approaches to meaning.

- **Type**: Single-page web application (SPA)
- **Tech Stack**: React 19, TypeScript, Redux Toolkit, Material-UI, Vite
- **Languages**: English, Polish (with i18next)
- **Deployment**: GitHub Pages
- **License**: MIT

## 📝 Documentation Conventions

### File Structure

- Main README at project root (`/README.md`)
- Documentation hub at `/docs/README.md`
- Detailed documentation in `/docs/*.md` files
- Source documentation in `/src/docs/*.md` (archived/internal)

### Version Control

Documentation is version-controlled with the main repository. All changes to documentation should be committed with clear, descriptive messages following the project's commit conventions.

### Keeping Documentation Current

- **Before implementing**: Review relevant documentation sections
- **After implementing**: Update documentation to reflect changes
- **When discovering patterns**: Document new patterns and conventions
- **For breaking changes**: Always update relevant documentation first

## 🤝 Contributing to Documentation

When contributing to ClarityAge documentation:

1. **Be clear and concise** - Aim for straightforward language
2. **Provide examples** - Code snippets and configuration examples
3. **Keep it current** - Update documentation alongside code changes
4. **Link appropriately** - Cross-reference related sections
5. **Consider the audience** - Write for the intended reader (developer, user, AI agent)
